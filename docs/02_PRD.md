## 📖 개요 및 목적 (Overview)

본 문서는 신발 쇼핑몰의 **상품 목록 페이지** 구현을 위한 요구사항, UI/UX 설계, 그리고 기능 명세를 정의합니다.
모바일 환경에서의 사용자 경험(UX)을 최우선으로 하며, 직관적인 상품 노출과 장바구니 접근성을 높여 구매 전환율을 극대화하는 것을 목적으로 합니다.

---

## 📝 고객 요구사항 (Requirements)

### 1. 모바일 최적화 (Mobile Optimization)
- **최우선 과제:** 웹 기반이지만 모바일 환경에서의 사용 편의성을 최우선으로 개발합니다.
- **이탈 방지:** 다양한 모바일 기기 해상도에서 화면 깨짐이나 비율 오류가 발생하지 않도록 반응형 처리를 철저히 합니다.

### 2. 상품 리스트 UI 개선 (UI Improvements)
- **이미지 중심 배치:** 텍스트(상품명 등)보다 **상품 사진(이미지)**이 눈에 띄도록 배치합니다.
- **디테일 강조:** 신발의 디자인과 디테일이 잘 보이도록 레이아웃을 구성합니다.
- **장바구니 접근성 강화:**
    - 화면 상단에 장바구니 아이콘을 고정 배치합니다.
    - **실시간 배지(Badge):** 장바구니에 상품을 담을 때마다 아이콘에 숫자 배지가 즉시 업데이트되어야 합니다.

### 3. 테스트 및 검수 (Testing)
- 내부 직원 및 이해관계자 피드백을 위해 접근 가능한 **테스트 URL**을 배포하고 공유해야 합니다.

## 📋 기능 및 컴포넌트 명세 (Functional Specs)

### 1. 레이아웃 (Layout)
전체 페이지의 공통 레이아웃 구조입니다.

- **Viewport 설정:** 모바일 기기 대응을 위한 `meta` 태그 및 `safe-area` 고려.
- **반응형 컨테이너:** 데스크톱에서 열람 시 너무 넓어지지 않도록 `max-width` (예: 480px ~ 768px) 설정 및 중앙 정렬 (`margin: 0 auto`).

### 2. 헤더 (Header)
사용자 네비게이션 및 장바구니 상태를 표시하는 상단 영역입니다.

- **컴포넌트:** `<Header />`
- **Sticky Position:** 스크롤 시에도 상단에 고정되어 언제든 접근 가능해야 함.
- **장바구니 아이콘:** 우측 상단 배치.
- **기능:**
    - **Badge 표시:** `cartCount` 상태값에 따라 빨간색 원형 배지에 숫자 렌더링.
    - **조건부 렌더링:** `cartCount === 0`일 경우 배지 숨김 처리.

### 3. 상품 리스트 (Product List)
상품들을 격자 형태로 나열하는 메인 콘텐츠 영역입니다.

- **컴포넌트:** `<ProductList />`
- **Grid Layout:** CSS Grid를 사용하여 한 줄에 2개의 상품을 배치합니다.
    ```css
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 16px; /* 또는 디자인 가이드에 따름 */
    ```
- **이미지 비율 고정:** 다양한 크기의 원본 이미지가 들어와도 레이아웃이 깨지지 않도록 설정합니다.
    - `aspect-ratio`: 정사각형(1/1) 또는 세로형(3/4) 등 고정 비율 적용.
    - `object-fit: cover`: 이미지가 영역을 꽉 채우도록 설정.

### 4. 상품 아이템 (Product Item)
개별 상품 카드를 구성하는 단위 컴포넌트입니다.

- **컴포넌트:** `<ProductItem />`
- **표시 정보:**
    - 상품 이미지 (클릭 시 상세 페이지 이동 가능)
    - 브랜드명 (보조 텍스트)
    - 상품명 (메인 텍스트)
    - 가격 (천 단위 콤마 포맷팅)
- **인터랙션:**
    - **장바구니 담기 버튼:** 상품 카드 내(보통 우측 하단)에 장바구니 아이콘 버튼 배치.
    - **Click Event:** 버튼 클릭 시 전역 상태 관리(Context/Store)의 `addToCart` 함수 호출하여 카운트 증가.

## 📂 폴더 구조 예시 (Directory Structure)

```bash
src/
├── components/
│   ├── layout/
│   │   └── Layout.jsx       # 모바일 중앙 정렬 컨테이너
│   ├── common/
│   │   └── Header.jsx       # 상단 고정 헤더 (장바구니 배지 포함)
│   └── product/

│       └── ProductItem.jsx  # 개별 상품 컴포넌트
├── pages/
│   └── ProductPage.jsx      # 메인 페이지 조합
└── App.jsx

🚀 실행 및 개발 방법

1. 프로젝트 설치
npm install
# 또는
yarn install

2. 개발 서버 실행
npm run dev
# localhost:3000 (또는 설정된 포트)에서 확인

3. 모바일 뷰 테스트
크롬 개발자 도구(F12) > Toggle Device Toolbar (Ctrl+Shift+M) 활성화 후 모바일 기기(iPhone SE, Pixel 등) 선택하여 확인.
