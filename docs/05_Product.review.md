# [SHOOKING_STORE] 요구사항 리뷰 및 기능 정의 (Product Review)

## 1. 문서 개요

본 문서는 고객사 이메일 요청사항과 전달받은 피그마(Figma) 시안을 바탕으로, 모바일 중심의 패션 쇼핑몰 서비스 구현을 위한 주요 요구사항과 UI/UX 기능을 정리한 문서입니다. 개발 착수 전, 고객사와의 최종 방향성 동기화를 목적으로 합니다.

---

## 2. 고객사 요청사항 (이메일 기반)

고객사 이메일 커뮤니케이션을 통해 파악한 핵심 비즈니스 및 서비스 요구사항입니다.

- **모바일 퍼스트(Mobile-First) 최적화:** 모바일 환경에서 상품을 빠르게 탐색하고 구매할 수 있는 직관적인 사용성 확보
- **상품 리스트 성능:** 다양한 패션/신발 상품 이미지가 매끄럽게 로딩되는 쾌적한 리스트 페이지 구현
- **구매 전환율 향상:** 상품 탐색 중 흐름이 끊기지 않고 즉시 장바구니에 담을 수 있는 동선 구축

---

## 3. 피그마(Figma) 시안 분석 및 UI 요구사항

전달해주신 피그마 디자인을 바탕으로 분석한 **모바일 UI 핵심 요소**입니다.

### 3.1. 상품 리스트 페이지 (Product List)

- **그리드 레이아웃:** 모바일 화면 기준 2열(2-Column) 상품 배치로 시인성 확보
- **상품 정보 노출:** 썸네일 이미지, 브랜드명, 상품명, 가격, 할인율 직관적 표기
- **상단 필터 및 정렬:** 카테고리 스와이프 필터(전체, 신발, 의류 등) 및 정렬 탭(신상품순, 가격순) 적용

### 3.2. 장바구니 담기 (Add to Cart)

- **리스트 내 빠른 담기:** 상품 썸네일 우측 하단에 '장바구니 아이콘' 버튼 배치
- **옵션 선택 바텀 시트 (Bottom Sheet):** 장바구니 아이콘 클릭 시, 페이지 이동 없이 화면 하단에서 옵션(사이즈/색상)을 선택할 수 있는 팝업 시트 제공
- **스낵바(Snackbar) 알림:** 장바구니 담기 성공 시, 상단 혹은 하단에 "장바구니에 추가되었습니다." 토스트 메시지 노출

## 4. 주요 기능 명세 (Feature Specifications)

| 기능 분류     | 세부 기능       | 설명                                                       | 비고                  |
| :------------ | :-------------- | :--------------------------------------------------------- | :-------------------- |
| **상품 탐색** | 상품 목록 조회  | 등록된 상품 데이터를 모바일 UI에 맞게 렌더링               | 무한 스크롤 적용 예정 |
|               | 카테고리 필터링 | 선택한 카테고리에 맞는 상품만 동적으로 필터링              |                       |
| **장바구니**  | 옵션 선택       | 장바구니 담기 시 사이즈, 색상 등 필수 옵션 선택 기능       | 바텀 시트 UI          |
|               | 장바구니 추가   | 선택한 상품과 옵션을 장바구니 데이터에 저장                |                       |
| **공통**      | 반응형 처리     | 모바일을 기준으로 하되, 웹 환경에서도 깨지지 않는 레이아웃 |                       |
